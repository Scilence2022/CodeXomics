Edit Mode Sequence Fix Test

Testing the fix for sequence object error when switching to Edit Mode

Problem Description

‚ùå Original Error

When clicking "Edit Mode" for the first time, the following error occurred:

SequenceUtils.js:193 Invalid sequence provided to VS Code editor: {COLI-K12: 'AGCTTTTCATTCTGACTGCAACGGGCAATATGTCTCTGTGTGGATTAAAA‚Ä¶TATTGAAAAAAATATCACCAAATAAAAAACGCCTTAGTAAGTATTTTTC'}

Root Cause: The sequence parameter was passed as an object {chromosome: sequence} instead of a string.

Fix Implementation

‚ùå Before (Buggy Code)

// In toggleDisplayMode() const sequence = this.genomeBrowser.currentSequence; // This is an object: {COLI-K12: "AGCTT..."} this.displayEnhancedSequence(chromosome, sequence); // Passes object instead of string

‚úÖ After (Fixed Code)

// In toggleDisplayMode() const sequenceData = this.genomeBrowser.currentSequence; if (chromosome && sequenceData && sequenceData[chromosome]) { // Extract the actual sequence string const sequence = sequenceData[chromosome]; this.displayEnhancedSequence(chromosome, sequence); }

Changes Made

  1. Fixed sequence extraction in toggleDisplayMode():
    • Changed const sequence = this.genomeBrowser.currentSequence
    • To const sequence = sequenceData[chromosome]
  2. Added better parameter validation:
    • Check that sequenceData[chromosome] exists
    • Extract string value before passing to display methods
  3. Enhanced debug logging:
    • Added console.log to show sequence type and length
    • Better error messages when validation fails
  4. Fixed variable name conflicts:
    • Renamed variables to avoid confusion between start/end parameters
    • Used editStart and editEnd for VS Code editor
