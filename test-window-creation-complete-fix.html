解决菜单系统冲突和文件加载问题

❌ 原始问题分析

问题 1: 菜单系统冲突

症状: 新打开的主窗口应用了Project Manager的菜单系统，而不是主窗口的菜单系统

原因:

  • 新窗口创建时的焦点管理时机不当
  • Project Manager的blur事件处理过于激进
  • 窗口焦点切换时菜单设置存在竞争条件

问题 2: 文件加载失败

症状: 主窗口没有打开点击的文件

原因:

  • 文件加载事件发送过早，窗口未完全初始化
  • 模块加载时机和文件加载时机不匹配
  • GenomeBrowser实例可能未完全准备好

✅ 解决方案

菜单系统修复:

// 改进新窗口的焦点管理 newMainWindow.once('ready-to-show', () => { newMainWindow.show(); newMainWindow.focus(); // 立即获得焦点 currentActiveWindow = newMainWindow; createMenu(); // 立即设置主窗口菜单 }); // 改进Project Manager的blur处理 projectManagerWindow.on('blur', () => { const mainWindows = BrowserWindow.getAllWindows().filter(win => win.getTitle().includes('Genome AI Studio') && !win.getTitle().includes('Project Manager') ); setTimeout(() => { const focusedMainWindow = mainWindows.find(win => win.isFocused()); if (focusedMainWindow) { currentActiveWindow = focusedMainWindow; createMenu(); // 恢复主窗口菜单 } }, 200); // 增加延迟确保焦点稳定 });

文件加载修复:

// 多阶段初始化确保完全准备 newMainWindow.webContents.on('did-finish-load', () => { if (reloadCount < maxReloads) { reloadCount++; newMainWindow.webContents.reload(); } else { // 等待DOM和模块完全初始化 setTimeout(() => { // 发送ping测试确保响应性 newMainWindow.webContents.send('ping-test'); setTimeout(() => { // 发送文件加载事件 newMainWindow.webContents.send('load-file', filePath); }, 500); }, 1500); // 扩展延迟确保完整初始化 } });

🚨 常见问题解决

1. 窗口创建但菜单错误:

  • 手动点击新窗口使其获得焦点
  • 检查是否有多个Project Manager窗口
  • 重启应用程序重置窗口状态

2. 文件加载失败:

  • 等待5秒再判断是否失败
  • 检查文件路径是否正确
  • 验证文件格式是否支持
  • 手动使用File→Open File重新加载

3. 控制台错误:

  • 查看是否有"require is not defined"错误
  • 检查模块加载失败信息
  • 验证preload脚本是否正确执行

4. 性能问题:

  • 关闭不必要的开发者工具窗口
  • 检查系统内存使用情况
  • 重启应用清除累积缓存
