🔧 Project Manager 修复验证

问题1: Save按钮总是弹出选择目录

修复内容
修改了 saveCurrentProject() 方法，现在会自动生成项目文件路径而不是弹出目录选择对话框。

修复前

if (!this.currentProject.projectFilePath) { return await this.saveProjectAs(); }

没有项目路径时总是调用另存为功能

修复后

if (!this.currentProject.projectFilePath) { const documentsPath = await window.electronAPI.getDocumentsPath(); const projectsDir = `${documentsPath}/Genome AI Studio Projects`; this.currentProject.projectFilePath = `${projectsDir}/${this.currentProject.name}.prj.GAI`; }

自动生成项目文件路径，直接保存

测试方法: 创建新项目后，点击Save按钮应该直接保存，不再弹出目录选择对话框。

问题2: 文件路径结构错误

修复内容
移除了多余的 data/ 子目录，文件现在直接存储在项目根目录的相应文件夹中。

修复前 (错误结构)

Genome AI Studio Projects/ ├── ProjectName.prj.GAI ├── ProjectName/ │ ├── Genomes/ (空) │ ├── Annotations/ (空) │ └── data/ │ ├── Genomes/ (实际文件) │ ├── Annotations/ (实际文件) │ └── Files/ (实际文件)

修复后 (正确结构)

Genome AI Studio Projects/ ├── ProjectName.prj.GAI └── ProjectName/ ├── Genomes/ (实际文件) ├── Annotations/ (实际文件) ├── Variants/ (实际文件) ├── Reads/ (实际文件) └── Analysis/ (实际文件)
修改的函数:
  • copyFileToProject - 移除了data子目录
  • moveFileInProject - 修正了移动路径
  • createNewProjectStructure - 使用正确的文件夹名称

功能3: 新增自定义数据文件夹

新功能特性
添加了创建自定义数据类型文件夹的功能，用户可以根据研究需要创建专门的数据目录。
  • 自定义文件夹名称 (如 Proteomics, Metabolomics)
  • 选择合适的图标 (🧪 🔬 📊 等15种选项)
  • 添加可选的描述信息
  • 自动在文件系统中创建对应目录
  • 集成到项目管理器界面
  • 支持拖拽文件到新文件夹
