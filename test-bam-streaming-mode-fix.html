Testing the fix for BAM files incorrectly using streaming mode instead of BAM mode

üêõ Problem Description

Before the fix, BAM files were incorrectly using streaming mode when they were larger than the streaming threshold (default 50MB). This caused:

  • BAM files being treated as SAM files
  • ReadsManager showing "mode: SAM" instead of "mode: BAM"
  • "No Reads in this region" errors due to incorrect parsing
  • Loss of BAM indexing benefits

‚úÖ Fix Applied

// OLD (BUGGY) CODE: const shouldUseStreaming = (fileSizeMB > streamingThreshold) || (extension === '.sam' && fileSizeMB > streamingThreshold); // NEW (FIXED) CODE: const shouldUseStreaming = (extension === '.sam' && fileSizeMB > streamingThreshold);

Key Changes:

  • Removed the general file size check that affected all file types
  • Now only SAM files use streaming mode when large
  • BAM files always use regular loading ‚Üí parseBAM() ‚Üí BamReader
  • Added better logging to show which mode is being used
