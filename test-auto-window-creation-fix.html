🔧 Auto Window Creation Fix

主窗口WebPreferences配置修复

❌ 原始问题

错误信息:

renderer-modular.js:2 Uncaught ReferenceError: require is not defined at renderer-modular.js:2:25

问题原因: 新创建的主窗口webPreferences配置与原始主窗口不一致

配置项 原始主窗口 新窗口（修复前） 状态
nodeIntegration true false 不匹配
contextIsolation false true 不匹配
enableRemoteModule true 未设置 缺失
webSecurity false 未设置 缺失
cache false 未设置 缺失

✅ 修复方案

1. 统一WebPreferences配置

修复前 (createNewMainWindow):

webPreferences: { nodeIntegration: false, contextIsolation: true, preload: path.join(__dirname, 'preload.js') }

修复后 (与原始主窗口一致):

webPreferences: { nodeIntegration: true, contextIsolation: false, enableRemoteModule: true, webSecurity: false, cache: false }

2. 添加完整初始化流程

  • ✅ 缓存清理逻辑
  • ✅ 强制重载机制
  • ✅ 窗口焦点管理
  • ✅ 菜单状态同步
  • ✅ 开发者工具自动开启

nodeIntegration true true 匹配
contextIsolation false false 匹配
enableRemoteModule true true 匹配
webSecurity false false 匹配
cache false false 匹配

🔧 技术实现细节

修改文件: src/main.js

修改函数: createNewMainWindow()

核心变更:

// 统一webPreferences配置 webPreferences: { nodeIntegration: true, // 允许渲染进程使用Node.js API contextIsolation: false, // 允许渲染进程访问Node.js环境 enableRemoteModule: true, // 启用remote模块 webSecurity: false, // 禁用web安全限制 cache: false // 禁用缓存 } // 添加完整初始化 + 缓存清理逻辑 + 窗口焦点管理 + 菜单状态同步 + 开发者工具配置

兼容性考虑:

  • 保持与现有主窗口代码100%兼容
  • 不影响其他工具窗口配置
  • 维持现有安全和性能特性

🚨 故障排除指南

如果仍然出现"require is not defined"错误:

  1. 重启应用程序，确保新配置生效
  2. 检查main.js中createNewMainWindow函数是否正确修改
  3. 验证webPreferences配置是否与原始主窗口一致
  4. 清除应用缓存和存储数据

如果窗口创建失败:

  1. 检查控制台错误日志
  2. 确认文件路径有效
  3. 验证IPC通信正常
  4. 检查窗口创建权限

如果功能按钮不可用:

  1. 等待所有模块加载完成
  2. 检查GenomeBrowser初始化状态
  3. 验证事件监听器注册
  4. 确认DOM元素正确创建
