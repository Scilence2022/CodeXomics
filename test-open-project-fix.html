问题分析与解决方案

原始错误: Error: File system access not available
位置: project-manager.html:1483
根本原因:
  • preload.js中缺少 readFile API暴露
  • 项目管理器期望 result.content，但main.js返回 result.data

实施的修复

1. 在preload.js中添加readFile API: 
// 添加到preload.js
readFile: (filePath) => ipcRenderer.invoke('read-file', filePath)
2. 修正project-manager.html中的数据字段引用: 
// 从 result.content 改为 result.data
await this.loadProjectFromText(result.data, filePath);
3. 确保IPC处理器正常工作:
  • ✅ main.js中已有 ipcMain.handle('read-file', ...)
  • ✅ 返回格式: { success: true, data: fileContent }
  • ✅ 支持大文件检查和错误处理

技术细节

文件读取流程:

  1. 用户选择 File → Open Project
  2. main.js 创建文件选择对话框
  3. 选中文件后，触发 onLoadProjectFromMenu 事件
  4. project-manager.html 调用 loadProjectFromFile(filePath)
  5. 通过 window.electronAPI.readFile(filePath) 读取文件
  6. preload.js 将请求转发给 main.js 的 'read-file' 处理器
  7. main.js 读取文件并返回 { success: true, data: content }
  8. 项目管理器解析内容并加载项目
