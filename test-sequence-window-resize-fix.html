üêõ Issue Description

Problem: When the window is too wide, the Sequence track window shows a white screen with errors:
  • SequenceUtils.js:667 Uncaught ReferenceError: viewEnd is not defined
  • Error: <svg> attribute viewBox: Trailing garbage, "0 0 2015 100%"
Fixes Applied:
  1. SequenceUtils.js: Added calculation of viewEnd in renderFullSequence() method
  2. TrackRenderer.js: Fixed SVG viewBox height calculation to use numeric values instead of percentage strings

üîß Fix 1: SequenceUtils viewEnd Variable

// Before (line 667 in renderFullSequence): this.addProteinTranslations(tempDiv, chromosome, subsequence, viewStart, viewEnd, annotations); // ‚ùå viewEnd was not defined // After: renderFullSequence(container, chromosome, subsequence, viewStart, annotations, operons, charWidth, optimalLineLength, sequenceSettings, featureLookup) { // Calculate viewEnd based on subsequence length const viewEnd = viewStart + subsequence.length - 1; // ... rest of method this.addProteinTranslations(tempDiv, chromosome, subsequence, viewStart, viewEnd, annotations); // ‚úÖ viewEnd is now properly calculated }
Status: Fixed - viewEnd is now calculated as viewStart + subsequence.length - 1

üîß Fix 2: SVG viewBox Height Calculation

// Before (multiple locations in TrackRenderer.js): svgContainer.setAttribute('viewBox', `0 0 ${containerWidth} ${svgContainer.getAttribute('height')}`); // ‚ùå getAttribute('height') could return "100%" string // After: const svgHeight = svgContainer.getBoundingClientRect().height || svgContainer.offsetHeight || defaultHeight; svgContainer.setAttribute('viewBox', `0 0 ${containerWidth} ${svgHeight}`); // ‚úÖ svgHeight is always a numeric value
Locations Fixed:
  • updateReadsTrackSVG() - default height: 150px
  • updateGCTrackSVG() - default height: 100px
  • updateOtherTracksSVG() for WIG tracks - default height: 30px
  • updateOtherTracksSVG() for variant tracks - default height: 30px
